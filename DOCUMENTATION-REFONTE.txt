0. RÉSUMÉ EN UNE PHRASE
-------------------------------------------------------------------------------
Le portfolio, basé sur un template gratuit générique (FreeFolio / HTML Codex),
a été entièrement refondu avec une identité éditoriale sur-mesure, un code
moderne sans dépendances lourdes, une meilleure accessibilité/performance, et
un référencement Google (SEO) complet.


-------------------------------------------------------------------------------
1. POINT DE DÉPART (AVANT)
-------------------------------------------------------------------------------
- Template gratuit « FreeFolio » de HTML Codex, look « déjà vu ».
- Stack lourd : Bootstrap 4.4.1 + jQuery + 6 librairies (owlcarousel, isotope,
  typed, waypoints, lightbox, easing).
- Problèmes constatés :
  * Métadonnées jamais personnalisées (« Free HTML Templates »), lang="en"
    alors que le contenu est en français.
  * HTML cassé dans la section Parcours (balises <p>/<div> non fermées).
  * Fautes : « INDESING », « HTML § CSS », « Qualitè ».
  * Images très lourdes (photo d'accueil : 1,5 Mo), attributs alt vides.
  * Formulaire de contact en PHP → NE FONCTIONNE PAS sur GitHub Pages
    (GitHub Pages ne fait pas tourner de PHP).
  * Liens sociaux vides (href="#").


-------------------------------------------------------------------------------
2. MÉTHODE — DEUX « SKILLS » DE DESIGN UTILISÉS
-------------------------------------------------------------------------------
La refonte s'appuie sur les deux skills installés dans le dossier .claude/ :

A) frontend-design  → LA DIRECTION CRÉATIVE
   Objectif : sortir du rendu « template » et créer une identité unique,
   ancrée dans le monde de Zita (relations publiques, communication, média/TV,
   Fianarantsoa).

B) ui-ux-pro-max   → LA QUALITÉ & L'IMPLÉMENTATION
   Objectif : appliquer une checklist priorisée (accessibilité, interaction,
   performance, typographie, responsive, animation).

Les deux sont complémentaires : l'un décide la direction, l'autre garantit une
mise en œuvre propre et accessible.

3. IDENTITÉ VISUELLE RETENUE — « LA RÉDACTION »
-------------------------------------------------------------------------------
Concept : une identité éditoriale + broadcast (télé/radio), qui reflète le
métier de communicante.

- Typographies (Google Fonts) :
  * Titrage  : Bricolage Grotesque  (caractère éditorial, non générique)
  * Corps    : Inter                (lisible, neutre)
  * Labels   : Space Mono           (voix « technique / broadcast »)

- Palette :
  * Papier chaud   #FBF8F3
  * Encre          #17161B
  * Bleu électrique#1B34E0  (primaire — crédible, « média »)
  * Corail         #FF4A1C  (accent « on air »)

- Élément signature : un BANDEAU DÉFILANT type « chyron » de chaîne TV
  (défile en continu sous l'accueil). C'est le seul effet « fort » ; tout le

- Structure éditoriale : sections numérotées façon rubriques de magazine

  a été RETIRÉE (ce n'est pas une séquence) mais CONSERVÉE sur le Parcours
  (vraie chronologie).

5. CHANGEMENTS TECHNIQUES
-------------------------------------------------------------------------------
- Suppression de Bootstrap, jQuery et des 6 librairies : le site est désormais
  100 % HTML/CSS/JS « vanilla », plus léger et plus rapide.

- Accessibilité (a11y) :
  * lang="fr", lien d'évitement (« Aller au contenu »)
  * focus visible au clavier, attributs aria-*, rôles
  * textes alternatifs (alt) réels sur les images
  * hiérarchie de titres correcte, cibles tactiles >= 44 px
  * respect de « prefers-reduced-motion » (animations coupées si demandé)

- Performance :
  * Photos compressées (voir §4), lazy-loading des images sous la ligne de
    flottaison, largeur/hauteur fixées (évite les sauts de mise en page / CLS)
  * Beaucoup moins de requêtes réseau (plus de CDN Bootstrap/jQuery/etc.)

- Fonctionnalités JS (js/refonte.js) :
  menu mobile, lien de navigation actif au défilement (scrollspy), apparition
  au défilement, filtre du portfolio, lightbox (agrandissement d'image),
  bouton « retour en haut », année du footer automatique, formulaire -> mailto.

8. RÉFÉRENCEMENT — CE QU'IL RESTE À FAIRE (ACTIONS MANUELLES)
-------------------------------------------------------------------------------
Le code ne suffit pas : il faut demander à Google d'indexer le site.

1) Google Search Console : https://search.google.com/search-console
   - Ajouter une propriété -> « Préfixe d'URL » -> https://zitafannie.github.io/
   - Vérifier (méthode « Balise HTML » : coller la balise fournie dans le
     <head>, puis pousser et cliquer « Vérifier »).
   - Menu « Sitemaps » -> soumettre : sitemap.xml
   - Menu « Inspection de l'URL » -> coller l'URL -> « Demander une indexation ».

2) Backlinks : mettre le lien https://zitafannie.github.io dans les bios
   LinkedIn / Instagram / Facebook (accélère l'indexation et le classement).

3) Liens sociaux : dans le footer ils pointent encore vers « # ». Les
   remplacer par les vraies URL et les ajouter en « sameAs » dans le JSON-LD.

Vérification : taper « site:zitafannie.github.io » dans Google. Si la page
apparaît, elle est indexée. Ensuite « Zita Fannie Andrianarivony » devrait la
faire ressortir (nom peu concurrentiel = bon positionnement).

9. LANCER LE SITE EN LOCAL
-------------------------------------------------------------------------------
C'est un site statique (aucun build). Ouvrir un terminal :

   cd /home/judi/dev/zitafannie.github.io
   python3 -m http.server 8000

Puis ouvrir dans le navigateur :  http://localhost:8000

Alternatives équivalentes :
   npx serve            (Node)
   php -S localhost:8000
   Extension « Live Server » de VS Code (rechargement automatique)
